# Exploration Phase

Use this stage first for any product-oriented request.

## Objectives

- Produce at least two viable product options.
- Capture evidence and assumptions for each option.
- Review nearby/local skills and note missing capabilities in skillsmith.
- Recommend one option and record one rejected option with reason.

## Required Outputs

- `.product/research/options.md`: options, assumptions, upside, downside
- `.product/research/external-skills-scan.md`: related skills reviewed and relevance
- `.product/research/gaps.md`: capability gaps, impact, suggested fixes
- `.product/research/recommendation.md`: selected option, rejected option, decision criteria

## Gap Taxonomy

Classify each gap as one of:
- missing role skill
- missing reference depth
- weak routing metadata
- weak artifact contract
- weak acceptance criteria

## Exit Criteria

- At least two options compared.
- At least three concrete gaps identified (or explicit "none" with reason).
- One recommendation with measurable success criteria for next stage.
