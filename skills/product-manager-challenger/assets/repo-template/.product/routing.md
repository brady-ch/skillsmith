# Item Routing

Use this decision tree when a tracked item needs more detail.

- If the problem, value, or scope is unclear, update `questions.md`.
- If completion can be tested, define it in `acceptance.md`.
- If execution has sequencing or dependency risk, update `delivery-plan.md`.
- If progress is stopped by an external dependency or unresolved decision, update `blockers.md`.
- If none of the above applies, keep the item lightweight with `summary.md` and `acceptance.md`.
