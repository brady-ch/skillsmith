# Reference Router

Read this file first. Then load only one additional reference unless the current task clearly needs both the repo setup and the item-detail routing rules.

## Route

- Need to challenge a request, expose missing logic, or force prioritization: load `intake-and-challenge.md`
- Need to create or repair the repo checklist and tracking system: load `tracking-system.md`
- Need to decide which per-item detail files to create or update: load `item-routing.md`

## Stop Condition

Stop loading references once you can:
1. state what does not make sense yet
2. define the next tracked item or decision
3. point to the correct repo file for the follow-up detail
