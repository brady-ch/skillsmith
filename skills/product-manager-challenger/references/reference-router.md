# Reference Router

Read first. Load one more unless task needs both repo setup and item routing.

## Route

- Need challenge, missing logic, priority pressure: load `intake-and-challenge.md`
- Need repo checklist/tracking setup or repair: load `tracking-system.md`
- Need per-item detail routing: load `item-routing.md`

## Stop Condition

Stop when answer can:
1. state what does not make sense yet
2. define the next tracked item or decision
3. point to the correct repo file for the follow-up detail
