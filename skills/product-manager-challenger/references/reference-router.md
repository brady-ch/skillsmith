# Reference Router

Read first. Load one more unless task needs both repo setup and item routing.

## Route

- Ambiguous intent, greenfield, decompose scope, discovery before challenge: load `discovery-wenyan.md`
- Lock scope, design approval before implementation, spec freeze: load `approval-gate-wenyan.md`
- Prioritize backlog, tradeoffs, what to build first: load `prioritization-wenyan.md`
- Approved scope to bite-sized implementation plan skeleton: load `handoff-to-plan-wenyan.md`
- Need challenge, missing logic, priority pressure: load `intake-and-challenge-wenyan.md`
- Need repo checklist/tracking setup or repair: load `tracking-system-wenyan.md`
- Need per-item detail routing: load `item-routing-wenyan.md`

## Stop Condition

Stop when answer can:
1. state what does not make sense yet
2. define the next tracked item or decision
3. point to the correct repo file for the follow-up detail
